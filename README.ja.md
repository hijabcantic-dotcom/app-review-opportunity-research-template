# アプリレビュー機会リサーチテンプレート

公開されている App Store と Google Play のレビューを、顧客インタビューの質問とプロダクト仮説に変えるための創業者向けテンプレートです。

目的は機能要望の一覧を作ることではありません。誰が、どの場面で、どの仕事を完了できず、どのような損失が生じたかという、繰り返されるワークフローの失敗を見つけます。その後に、インタビューすべきかを判断します。

## 45分で使う方法

1. 同じユーザーの仕事に近い競合を一つ選びます。
2. 最近の低評価レビューを20から50件集めます。
3. 有用な原文を [`research-template.md`](research-template.md) に記録します。
4. 仕事、失敗、結果が実質的に同じ場合だけを一つのクラスタにまとめます。
5. 各クラスタのためにインタビュー質問を一つ書きます。レビューは手掛かりであり、需要の証明ではありません。

繰り返し、具体的な状況、明確なコストをすべて持つ問題を優先してください。近い競合2から4件でも確認し、複数の製品で繰り返されるパターンを深掘りします。

より大きなサンプルには、[Review2Idea](https://review2idea.com/ja/free-app-review-analysis-tool?utm_source=github&utm_medium=referral&utm_campaign=app_review_research_template) が公開レビューを根拠付きの課題クラスタにまとめます。元レビューを保持し、実際のユーザーインタビューで仮説を検証してください。
